0.7.0 Agent Token Saver

Local-first asset handoff for Phaser, Godot, LOVE2D, PixiJS, and Canvas with compact agent views, read-only wiring maps, reports, fixtures, recipes, MCP metadata, and P4 media metadata.

What It Is

openRender 0.7.0 documents the Agent Token Saver layer on top of the local media-to-engine Developer Kit: compact LLM handoff context, read-only wiring maps, compact table output, safe agent instruction install, deterministic adapter output, recipes, fixtures, report export, local gallery metadata, MCP metadata, and P4 media metadata contracts.

The compiler takes an existing local image and produces compiled PNG output, install plans, helper files, verification results, frame preview sheets, local reports, and rollback records.

The Problem

AI agents can write game code and receive generated images, but raw generated media rarely drops cleanly into a playable project. It needs normalization, frame validation, project-specific paths, helper code, preview, report, and a rollback boundary.

How openRender Helps

local image file
-> project scan
-> media contract
-> deterministic PNG artifact
-> project install plan
-> local files with snapshots
-> verify and report
-> agent uses generated paths/helpers
-> rollback remains available

Adapter Support

  • Asset paths: place compiled media where the target project can load it.
  • Manifest files: expose stable IDs, dimensions, frame metadata, and load paths.
  • Helper files: provide project-facing code for sprite frame sets and animation setup.

Implemented Agent Surfaces

  • Agent-safe CLI: context, schema, plan sprite, install-agent, explain, diff, compact JSON views, read-only wire maps, and JSON-first output.
  • Adapter registry: Phaser, Godot, LOVE2D, PixiJS, and Canvas share contract validation, install plans, helper generation, verification, and fixture coverage.
  • Image handoff quality: alpha diagnostics, detect-frames, sprite invariants, normalize presets, frame preview sheets, Pixi spritesheet JSON, and Canvas draw helpers.
  • Contributor base: tracked schemas, recipe validation, adapter scaffolding, fixture capture, compatibility matrix, and at least two golden fixtures per built-in target.
  • Report system: local reports, explicit report export, local gallery metadata, verification details, helper code, diffs, and rollback commands.
  • Agent/MCP surface: a local JSON-only MCP metadata package with context, install-agent, tools, resources, and prompts for supported targets.
  • P4 media metadata: schema-backed audio, atlas/tileset, UI asset metadata and runtime smoke availability checks.

Agent Output

context --json --compact provides a short project handoff. context --json --wire-map returns read-only wiring candidates. verify, report, explain, and diff support --compact JSON views with status, next actions, rollback information, and compact tables.

Version History

  • 0.4.0: adapter registry, PixiJS, Canvas, MCP metadata, agent init, and recipe substrate.
  • 0.5.0: adapter scaffold, fixture capture, report export/gallery, and stronger failure guidance.
  • 0.6.1: P4 media metadata contracts, runtime smoke availability checks, and expanded QA coverage.
  • 0.7.0: compact agent output, read-only wire-map, and compact table output for verify/report/explain/diff.

Detailed release notes are maintained in RELEASES.md and docs/release-history.html.

Safety And Privacy

The default workflow stays inside the target project, avoids network calls, does not require an account, and records installed files so the openRender change can be rolled back.
