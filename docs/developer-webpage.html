A small local loop

The page should show exactly what the POC does: initialize local state, compile a sprite, then verify and report the run.

01

Initialize

Create config and `.openrender` state directories inside the target project.

openrender init
02

Compile and install

Turn a raw image into an installed Phaser asset and helper files.

openrender compile sprite --from tmp/slime_raw.png --id enemy.slime.idle --frames 6 --frame-size 64x64 --install
03

Verify and report

Check local files and write an HTML/JSON report for review.

openrender verify --run latest && openrender report --run latest --open

Compiler pieces, not a playground

openRender is not an image generator. It is the deterministic layer between generated media and engine-ready project files.

Scan

Detect Vite, Phaser, source root, asset root, and local state.

openrender scan

Compile

Read image metadata, validate frame dimensions, and write artifacts.

.openrender/artifacts

Install

Copy assets and generate Phaser manifest or animation helpers.

public/assets

Rollback

Restore snapshots or remove files created by an install.

openrender rollback

What changes on disk

The trust model is visible files. No account, no billing, no cloud API, and no provider call are needed for the v0.1 path.

Narrow by design

v0.1 proves one local path before expanding into more engines, media types, or agent integrations.

Included

  • Vite + Phaser projects
  • Image assets
  • Transparent sprites and sprite frame sets
  • Local CLI workflow
  • HTML and JSON reports
  • Rollback from install snapshots

Not included

  • Account or billing
  • Cloud API or hosted playground
  • Model provider calls
  • Audio or video compiler
  • Full MCP server
  • Godot, Pixi, Unity, or Canvas adapters
