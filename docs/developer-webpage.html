Local-first image-to-Phaser POC

openRender

Local-first media-to-engine compiler for agentic game development. Turn raw generated images into Phaser-ready assets, codegen, local reports, and rollbackable project changes.

local CLI 
openrender compile sprite \
  --from tmp/slime_raw.png \
  --id enemy.slime.idle \
  --frames 6 \
  --frame-size 64x64 \
  --install \
  --json

Developing this repository? Use pnpm dev:cli with the same arguments.

Run it inside your project

Start from a Vite + Phaser project, then let the CLI create local state, compile a sprite, and write an inspectable report.

1

Initialize local state

Create config and local run directories. No account or cloud setup is required.

openrender init
2

Compile and install

Convert a raw image into a Phaser-ready asset, manifest, and animation helper.

openrender compile sprite --from tmp/slime_raw.png --id enemy.slime.idle --frames 6 --frame-size 64x64 --install
3

Verify and report

Check local files and open the generated HTML report for inspection.

openrender verify --run latest && openrender report --run latest --open

From generated media to playable project files

The v0.1 loop is intentionally narrow: image input, deterministic local processing, Phaser output, report, and rollback.

Scan

Detect project shape, package manager, Vite, Phaser, source root, and asset root.

openrender scan

Compile

Read image metadata, validate frames, and write a compiled artifact.

.openrender/artifacts

Install

Copy the asset and write Phaser manifest or animation helper files.

public/assets

Verify

Check artifact presence, installed files, and image dimensions.

openrender verify

Report

Write local HTML and JSON records that humans and agents can inspect.

.openrender/reports

Rollback

Restore snapshots or delete files created by install.

openrender rollback

Snapshots before install

Install records destination files before writing so rollback can restore or delete local changes.

Reports stay local

HTML previews and JSON run records are written under `.openrender`; no project files are uploaded.

Reports for humans and agents

Every run leaves a record. A developer can inspect the HTML report, and an agent can read the JSON run state before deciding whether to iterate or roll back.

v0.1 scope is intentionally narrow

This first page should be honest about the POC. It proves the local compiler loop before adding accounts, hosted services, or more engines.

Included

  • Vite + Phaser projects
  • Image assets
  • Transparent sprites
  • Sprite frame sets
  • Local CLI workflow
  • HTML and JSON reports
  • Rollback from install snapshots

Not included

  • Account or billing
  • Cloud API or hosted playground
  • Model provider calls
  • Audio or video compiler
  • Full MCP server
  • Godot, Pixi, Unity, or Canvas adapters
  • Automatic scene patching
